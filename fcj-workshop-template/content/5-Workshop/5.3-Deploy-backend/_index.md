This section guides users through building and deploying the AWS BILLO backend using AWS SAM and AWS CloudFormation.

The backend deployment creates the main AWS serverless resources, including Cognito, API Gateway, Lambda, DynamoDB, S3, CloudWatch Logs, and related IAM permissions.

---

## Step 1: Open the Backend Folder

Open PowerShell and go to the backend folder:

```powershell
cd C:\Users\admin\source\repos\AWS_BILLO\backend
```

Check the main backend files:

```powershell
dir
```

Make sure the folder contains:

```text
template.yaml
samconfig.toml
package.json
src
tests
```

---

## Step 2: Install Backend Dependencies

Install Node.js dependencies:

```powershell
npm install
```

This command installs the packages required by the Lambda functions and backend tests.

---

## Step 3: Review the SAM Template

Open the following file in Visual Studio Code:

```text
backend/template.yaml
```

This file defines the AWS resources used by the backend, such as:

- Cognito User Pool and App Client
- Cognito User Pool Groups
- API Gateway
- Lambda functions
- DynamoDB Main Table
- DynamoDB Idempotency Table
- S3 Upload Bucket
- CloudWatch Logs
- IAM roles and permissions

The backend is managed as Infrastructure as Code through AWS SAM and CloudFormation.

---

## Step 4: Build the Backend

Run:

```powershell
sam build
```

This command builds the Lambda functions and prepares the deployment artifacts.

If the build cache causes issues, run:

```powershell
sam build --no-cached
```

Expected result:

```text
Build Succeeded
```

---

## Step 5: Deploy the Backend

Deploy the backend using the existing SAM configuration:

```powershell
sam deploy
```

For faster deployment without repeated confirmation prompts, run:

```powershell
sam deploy --no-confirm-changeset --no-fail-on-empty-changeset
```

Expected stack information:

```text
Stack name: wallet-app-backend-dev
Region: ap-southeast-1
```

SAM deploys the backend through AWS CloudFormation.

---

## Step 6: Verify CloudFormation Stack

After deployment, open the AWS Management Console and go to:

```text
CloudFormation > Stacks
```

Find the stack:

```text
wallet-app-backend-dev
```

Check that the stack status is:

```text
CREATE_COMPLETE
```

or:

```text
UPDATE_COMPLETE
```

![CloudFormation Stack](/images/5-Workshop/cloudformation-stack.png)
If the stack fails, open the **Events** tab to identify the failed resource.

---

## Step 7: Verify API Gateway Endpoint

The backend API endpoint for the development environment is:
![API Gateway Endpoint](/images/5-Workshop/api-gateway-endpoint.png)

```text
https://zsqkp5vpb9.execute-api.ap-southeast-1.amazonaws.com/dev
```

This endpoint is used by the Flutter frontend and Admin Web to call backend APIs.

The endpoint should also be configured inside:

```text
frontend/config/dev.json
```

---

## Step 8: Verify DynamoDB Tables

Open the AWS Management Console and go to:

```text
DynamoDB > Tables
```

Check the main table:

```text
wallet-app-main-dev
```

The main table stores data such as:

- User profiles
- Wallets
- Merchant applications
- Stores
- Products or services
- Tables
- Orders
- Bills
- Payment sessions
- Transactions

The project also uses an idempotency design through DynamoDB to prevent duplicate processing for transfers and payments.

---

## Step 9: Verify S3 Bucket

Open:

```text
Amazon S3 > Buckets
```

Check the upload bucket created by the backend stack.

The S3 bucket is used to store:

- Business registration documents
- Store images
- Product or service images

Files are uploaded through pre-signed URLs generated by the backend.

---

## Step 10: Verify Cognito User Pool

Open:

```text
Amazon Cognito > User pools
```

Check the User Pool ID:

```text
ap-southeast-1_AKc39KB4L
```

The User Pool handles:

- Phone number registration
- OTP confirmation
- Login
- JWT token issuing
- User groups for Customer, Merchant, and Admin

Important note: SMS OTP depends on AWS SMS sandbox or production SMS configuration. In sandbox mode, only verified phone numbers may receive OTP messages.

---

## Step 11: Verify Lambda and CloudWatch Logs

Open:

```text
AWS Lambda > Functions
```

Check that the backend Lambda functions were created successfully.

Then open:

```text
CloudWatch > Log groups
```

Check the log groups for Lambda functions. These logs are useful for debugging API errors, authentication problems, payment issues, and data validation errors.

---

## Common Deployment Issues

### AWS credentials are not configured

Run:

```powershell
aws configure
```

Then check:

```powershell
aws sts get-caller-identity
```

### SAM build fails

Try:

```powershell
sam build --no-cached
```

Also check Node.js version:

```powershell
node -v
```

### CloudFormation stack rollback

Open:

```text
CloudFormation > Stacks > wallet-app-backend-dev > Events
```

Read the failed event and fix the related resource or permission.

### OTP is not received

If Cognito SMS is still in sandbox, use a verified phone number or configure production SMS access.
