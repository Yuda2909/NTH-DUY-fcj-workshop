Before starting this workshop, prepare the required AWS account, local tools, source code, and project configuration.

---

## AWS Account Requirements

You need an AWS account with permission to create and manage the following resources:

- Amazon Cognito User Pool
- Amazon API Gateway
- AWS Lambda
- Amazon DynamoDB
- Amazon S3
- Amazon CloudWatch Logs
- AWS CloudFormation stacks
- IAM roles and policies generated by AWS SAM

The project is deployed in the following AWS Region:

```text
ap-southeast-1
```

This is the Singapore Region.

## Local Development Tools

Install the following tools on your computer:

| Tool               | Purpose                                                  |
| ------------------ | -------------------------------------------------------- |
| AWS CLI            | Configure AWS credentials and interact with AWS services |
| AWS SAM CLI        | Build and deploy the serverless backend                  |
| Node.js 22.x       | Run and package Lambda backend code                      |
| npm                | Install backend and Admin Web dependencies               |
| Flutter SDK        | Run the Flutter frontend                                 |
| Git                | Clone and manage source code                             |
| Visual Studio Code | Edit and manage the project files                        |

---

## Check AWS CLI

Open PowerShell and run:

```powershell
aws --version
```

Then configure AWS credentials:

```powershell
aws configure
```

Enter the required information:

```text
AWS Access Key ID
AWS Secret Access Key
Default region name: ap-southeast-1
Default output format: json
```

Check the current AWS identity:

```powershell
aws sts get-caller-identity
```

If the command returns your AWS account ID and user ARN, the AWS CLI is configured successfully.

---

## Check AWS SAM CLI

Run:

```powershell
sam --version
```

If SAM CLI is installed correctly, the terminal will display the installed version.

---

## Check Node.js and npm

Run:

```powershell
node -v
npm -v
```

The backend uses Node.js 22.x for Lambda functions.

---

## Check Flutter

Run:

```powershell
flutter doctor
```

Make sure Flutter is installed and Chrome is available for web testing.

Then check connected devices:

```powershell
flutter devices
```

Chrome should appear in the device list.

## Backend Configuration Files

The backend folder should contain:

```text
backend/
├── template.yaml
├── samconfig.toml
├── package.json
├── src/functions/
├── src/shared/
└── tests/
```

Important files:

| File             | Purpose                                  |
| ---------------- | ---------------------------------------- |
| `template.yaml`  | Defines AWS resources for SAM deployment |
| `samconfig.toml` | Stores deployment configuration          |
| `package.json`   | Stores backend dependencies and scripts  |
| `src/functions/` | Contains Lambda functions                |
| `src/shared/`    | Contains shared backend code             |

## Frontend Configuration

The Flutter app uses a configuration file to connect to the deployed AWS backend:

```text
frontend/config/dev.json
```

This file should include values such as:

```json
{
  "API_BASE_URL": "https://zsqkp5vpb9.execute-api.ap-southeast-1.amazonaws.com/dev",
  "AWS_REGION": "ap-southeast-1",
  "COGNITO_USER_POOL_ID": "ap-southeast-1_AKc39KB4L"
}
```

The actual key names may depend on the project implementation. Make sure the values match the deployed AWS backend.

---

## Important Notes

Cognito OTP SMS depends on AWS SMS configuration. If the account is still in SMS sandbox, only verified phone numbers may receive OTP messages.

The frontend and Admin Web are currently run locally. This workshop does not include AWS Amplify, Amazon S3 static hosting, or Amazon CloudFront.

Before running the workshop, make sure the AWS account has enough permissions and that AWS Budget alerts are configured to avoid unexpected costs.
