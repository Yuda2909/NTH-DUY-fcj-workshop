### Week 8 Achievements:

- Completed the initial analysis of the AWS BILLO project topic and clarified the system direction.

- Defined the three main roles of the system:
  - Customer
  - Merchant
  - Admin

- Identified the MVP scope and prioritized the core features for the first version of the project.

- Analyzed the main Customer functions:
  - Account registration and login
  - Wallet balance display
  - Money transfer
  - QR payment
  - Transaction history
  - Digital bill preview

- Analyzed the main Merchant functions:
  - Business registration
  - Store profile management
  - Product or service management
  - Table management
  - Order creation
  - QR-based payment support

- Analyzed the main Admin functions:
  - Review merchant applications
  - Approve or reject business registration
  - Manage user roles and merchant access

- Completed the initial user flows and business flows for registration, authentication, merchant approval, store management, order creation, and QR payment.

- Researched suitable AWS services for the system, including:
  - Amazon Cognito for authentication and user groups
  - AWS Lambda for business logic
  - Amazon API Gateway for API management
  - Amazon DynamoDB for NoSQL data storage
  - Amazon S3 for image and document storage
  - Amazon CloudWatch for logging and monitoring

- Compared serverless architecture with EC2/VPC-based architecture and selected a cost-optimized serverless direction for the MVP.

- Completed the initial AWS architecture design, wireframe notes, user flow, and business flow to prepare for implementation in the following weeks.
